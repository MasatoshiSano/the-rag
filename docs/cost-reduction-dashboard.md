## 1. 目的（Why）

2025 年度コスト削減目標 **6,000 万円**（売上高比 1.5%）の進捗を、現行 Excel 管理ツールに代わってリアルタイムに可視化する。

### 背景
- 2024 年度実績 4,200 万円 / 目標 5,000 万円（達成率 84%、未達要因は材料価格高騰）
- 議事録「4. 進捗管理方法」で **「コスト削減管理システム（Excel）を刷新、ダッシュボード化」** が決定事項として明文化された
- 月次（第2月曜）報告会／四半期役員レビューの帳票作成負荷を削減し、経理 1 名工数 / 月 を 0.3 人月に圧縮する
- 経理課宿題「新管理帳票フォーマット」期限 **2025-01-20** に整合させる

### スコープ（本書の対象）
- ダッシュボードの要件・画面設計・データモデル・データ源接続方針
- 役員／部門長／施策担当者向けの3階層ビュー定義

### スコープ外
- BI ツール選定の最終確定（候補は§7）
- 認証基盤の新設（既存社内 SSO に乗せる前提）
- 製造 MES／販売管理側のスキーマ変更

## 3. 必要メトリクス（材料費・労務費・経費の3軸）

### 3.1 材料費（目標 3,000 万円）
| メトリクス | 計算定義 | 主要な切り口 |
|---|---|---|
| 集中購買による価格削減額 | 旧単価 × 当期数量 − 新単価 × 当期数量 | サプライヤー / 品目 / 月 |
| 代替材切替による削減額 | 標準単価差 × 切替後数量 | 品目 / 工程 / 月 |
| 歩留まり改善額 | (基準歩留 − 実績歩留) × 標準材料費 | 工程 / 品目 / 月 |
| 在庫金利削減額 | (基準平均在庫 − 実績平均在庫) × 在庫金利率 | 倉庫 / 品目グループ / 月 |

### 3.2 労務費（目標 1,500 万円）
| メトリクス | 計算定義 | 主要な切り口 |
|---|---|---|
| 段取り時間短縮効果 | (基準段取時間 − 実績段取時間) × 賃率 | 製造係 / 設備 / 月 |
| 多能工化による工数削減 | 配置最適化前後の工数差 × 賃率 | 製造係 / 月 |
| 残業時間削減額 | (基準残業 − 実績残業) × 残業単価 | 部門 / 月 |

### 3.3 経費（目標 1,500 万円）
| メトリクス | 計算定義 | 主要な切り口 |
|---|---|---|
| 電力使用量削減額 | (基準 kWh − 実績 kWh) × 電力単価 | 計器 / 棟 / 月 |
| 消耗品費削減額 | 基準消耗品費 − 実績消耗品費 | 費目 / 部門 / 月 |
| 外注加工内製化効果 | 外注単価 × 内製化数量 − 内製変動費 | 品目 / 月 |
| 物流費見直し効果 | 旧物流契約 − 新物流契約 | 路線 / 月 |

### 3.4 共通 KPI
- 累計削減額（YTD） / 年度目標達成率
- 月次計画 vs 実績差異
- 前年同月比（YoY）
- 進捗ステータス（青=計画線±5% / 黄=±10% / 赤=超過遅延）

## 4. 表示形式（部門別・月別・前年比）

### 4.1 画面構成
1. **役員サマリビュー（Top）**
   - ゲージ：年度目標 6,000 万円に対する累計実績
   - 3軸スタックバー：材料費・労務費・経費の月次削減実績 vs 計画
   - 達成率ヒートマップ（部門 × 月）

2. **部門別ドリルダウン**
   - タブ: 購買 / 製造 / 製造技術 / 経理
   - 折れ線：月次実績 / 計画 / 前年同月実績
   - 棒グラフ：施策別寄与額（積み上げ）
   - 表：施策ごとに 計画額・実績額・達成率・責任者・期限・ステータス

3. **施策トラッカー**
   - カンバン：未着手 / 進行中 / 効果検証中 / 完了
   - 担当者ビュー（マイ施策のみフィルタ）

### 4.2 切り口
- 期間: 月 / 四半期 / 年度（YTD）
- 比較: 計画 / 前年実績 / 過去3年平均
- ディメンション: 部門・施策カテゴリ・サプライヤー・品目・工程

### 4.3 出力
- PDF エクスポート（役員報告会用、A3 1 枚）
- CSV エクスポート（経理の二次集計用）

## 5. データ源（既存社内システム想定）

| データ種別 | 取得元 | 連携方式 | 頻度 |
|---|---|---|---|
| 製造原価実績（材料・労務・経費） | 会計システム（勘定奉行 想定） | 夜間バッチ CSV → DWH | 日次 |
| 購買単価・発注実績 | 購買管理システム | API or DB レプリカ | 日次 |
| 生産数量・歩留・段取り時間 | MES / 生産日報 | DB 直連携 | 日次 |
| 残業時間・出勤工数 | 勤怠システム | CSV 連携 | 日次 |
| 在庫金額・回転 | 在庫管理システム | DB 直連携 | 日次 |
| 電力使用量 | デマンド監視システム | API（議事録の新規導入分） | 時次→日次集約 |
| 物流費 | 物流会社請求データ | 月次 CSV 取込 | 月次 |
| 施策マスタ・計画値 | 本ダッシュボードの管理画面 | 手動入力（経理） | 都度 |

### 5.1 アーキテクチャ概観
```
[基幹各システム] --(ETL: 日次)--> [DWH (PostgreSQL or Snowflake)]
                                         |
                                  [Semantic Layer]
                                         |
                              [BI: Metabase / Looker Studio]
                                         |
                                [社内 SSO 越しに参照]
```

### 5.2 データモデル（論理）
- `fact_cost_actual`（年月, 部門, 費目, 金額）
- `fact_initiative_progress`（施策ID, 年月, 計画額, 実績額, ステータス）
- `dim_initiative`（施策ID, カテゴリ, 責任者, 開始月, 終了月, 目標額）
- `dim_department`, `dim_period`, `dim_account`

## 6. 非機能要件

| カテゴリ | 要件 |
|---|---|
| 性能 | 役員サマリ初回表示 3 秒以内（DWH 直叩きでなく集計テーブル経由） |
| 鮮度 | 前日実績を翌営業日 9:00 までに反映 |
| 権限 | 役員=全件閲覧 / 部門長=自部門+全社サマリ / 担当者=自施策のみ編集 |
| 監査 | 実績入力履歴を 7 年保持（税務監査要件に合わせる） |
| 可用性 | 平日 8:00-20:00 で 99% / 計画停止は月次報告会前後を避ける |

## 7. BI ツール候補（後続 Issue で確定）

| 候補 | 採用根拠の方向性 | 留意点 |
|---|---|---|
| Metabase（OSS） | 既に社内 PoC あり、コスト最小 | 大規模ジョイン性能 |
| Looker Studio | Google ライセンスを既に保有 | オンプレ DWH 接続要検討 |
| PowerBI | 経理が Excel 連携を希望 | ライセンス費 |

## 9. リスクと対応

| リスク | 影響 | 対応 |
|---|---|---|
| 代替材切替の品質評価遅延（議事録明記） | 材料費 800 万円分の効果計上が後ろ倒し | 評価ステータスを施策トラッカーで可視化、計画値を四半期で見直し |
| 材料価格高騰（2024 未達要因の再来） | 削減効果が市況で相殺 | 価格変動要因を効果額から分離表示 |
| 入力遅延 | 月次報告に間に合わない | 締め日 −2 営業日にリマインド自動送付 |

## 10. オープン論点（次回会議に持ち込み）

- 効果額の二重計上ルール（例: 内製化と労務費の按分）
- 為替・原材料市況の影響をダッシュボード上で「コスト削減効果」から控除するか
- BI ツールの最終決定者と決裁プロセス
