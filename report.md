# RAG Phantom - 社内ナレッジRAG検索システム 開発報告

> **ビジョン：製造現場の知識を、誰でも・すぐに・正確に引き出せる世界をつくる**

---

## 1. 現在の課題（Problem）

製造現場における社内ナレッジの活用で日常的に発生している課題を整理しました。

### 1-1. 社内文書が散在し、必要な情報にたどり着けない

製造マニュアル、トラブル対応記録、安全衛生ガイドラインなどの社内文書が、共有フォルダ・Wiki・個人PCに分散しています。ファイル名や格納場所に統一ルールがなく、「あの資料どこだっけ？」が頻発。結果として、過去に解決済みの問題を一から調べ直す工数が発生しています。

### 1-2. 検索しても欲しい情報がヒットしない

既存のファイル検索はキーワード完全一致が前提です。「モータ組立43号のラインコードは？」と聞きたくても、文書中の表記が「KJMA43」や「モータ組立#43」など揺れがあり、検索で見つかりません。

```
現状の情報アクセス
├── 共有フォルダ/   ← 階層が深く、命名規則がバラバラ
├── 社内Wiki/       ← 古い情報が更新されずに残存
├── 個人PC/         ← 属人化した知識が共有されない
└── → 「探す時間 > 読む時間」になっている
```

### 1-3. 製造データベースへの問い合わせにSQL知識が必要

製造実績・品質検査・トラブル履歴などはOracle DBに蓄積されていますが、参照にはSQL文の作成が必要です。現場のオペレーターや管理者が気軽にデータを確認できる手段がありません。

## 2. RAG Phantomでこう変わる（Solution）

| # | 現状（BEFORE） | 導入後（AFTER） |
|---|---|---|
| 1 | 社内文書が散在し、検索に時間がかかる | **ナレッジベース**にドキュメントを集約し、AIが即座に回答 |
| 2 | 共有フォルダを手動で探し回る | **フォルダパス登録**で、パスを指定するだけでAIが中身を検索 |
| 3 | 複数ファイルを開いて突き合わせ | **AIエージェント検索（深掘りモード）**がファイル探索→検索→読み込みを自律的に実行 |
| 4 | 表記ゆれでキーワード検索がヒットしない | **ユーザー辞書＋セマンティック検索**で意味ベースの検索を実現 |
| 5 | 誰が何を知っているか分からない | **ユーザーメモリ**が所属・担当を学習し、パーソナライズ回答 |
| 6 | 製造DBの参照にSQLが必要 | **自然言語クエリ**で「先月の不良率は？」と聞くだけでデータ取得（今後拡張） |

### ナレッジベースによる情報整理

```
RAG Phantom のナレッジ管理
└── ナレッジベース（KB）
    ├── 技術               ← 技術資料 25件
    ├── 安全衛生ガイドライン  ← 安全基準 2件
    ├── 製造マニュアル       ← 作業手順 1件
    ├── フォルダテスト       ← ローカルフォルダ連携（パス指定のみ）
    └── 要領書とか          ← 共有フォルダ連携（パス指定のみ）
```

**2つの登録方法**を提供:
- **アップロード方式**: ドキュメントをアップロード→ベクトル化→意味検索（高精度）
- **フォルダパス指定方式**: Windowsフォルダパスを登録するだけ。アップロード不要で、AIが検索時に直接ファイルを読み取る（手軽）

## 4. 主要機能の詳細

### 4-1. RAGチャット — ナレッジベースとの対話

ユーザーが自然言語で質問すると、関連ドキュメントを検索し、AIが根拠に基づいた回答を生成します。

```
入力: 「AWSのチャットで30秒で通信が切れるのはどう対処したらいい？」
  ↓
処理: クエリ解析 → 拡張クエリ3件生成 → Qdrant検索 → RRF統合 → Claude回答生成
  ↓
出力: 回答（SSEストリーミング）＋ 参照元ドキュメント一覧 ＋ 未登録用語の候補
```

**回答モード切替**: 「シンプル」で箇条書き要約、「詳細」で段落形式の網羅的回答。max_tokensも512/4096で制御。

**検索の高度化**:
- **ハイブリッド検索**: 密ベクトル＋スパースベクトルをRRFで統合
- **クエリ拡張**: AIが3件の言い換えクエリを自動生成し検索漏れを防止
- **ユーザー辞書**: 「モータ組立43号」→「KJMA43」のような表記ゆれを自動変換
- **リランク**: Cohere Rerank v3.5で検索結果を再順位付け（オプション）

### 4-2. ドキュメント管理 — アップロードから検索可能まで全自動

```
入力: PDF / Excel / Word / Markdown / CSV / 画像 など12形式対応
  ↓
処理: Markdown変換 → AIタグ付け → ユーザー確認 → チャンク分割 → ベクトル化
  ↓
出力: Qdrantに12,753チャンクとして格納済み → チャットから即検索可能
```

**4段階パイプライン**:
1. **変換（Converting）**: 各形式をMarkdownに統一。PDF→pymupdf4llm、PPTX→pptx2md等
2. **タグ付け（Tagging）**: Claude Sonnet 4.5が文書を分析し、拠点・ライン・工程・カテゴリ等を自動推定
3. **チャンク分割（Chunking）**: 最大512トークン、見出し階層に基づいた親子関係を維持
4. **インデックス化（Indexing）**: Cohere Embed v3でベクトル化し、Qdrantに格納

### 4-3. ユーザーメモリ — 対話から学ぶパーソナライズ

```
入力: 「私はソ企の佐野です。担当ラインはモータ組立43号です」
  ↓
処理: 会話からユーザー情報を自動抽出 → メモリに永続化
  ↓
出力: 次回以降「いつものラインのラインコードは？」→「KJMA43です」と即答
```

Google Geminiの「自分について」機能と同様、会話の中からユーザーの所属・担当・関心事を自動学習します。手動登録も可能です。

### 4-4. セッション全文検索 — 過去の対話を横断検索

SQLite FTS5による高速全文検索で、過去のすべてのチャットセッションから関連する会話を検索できます。「あのとき聞いた回答をもう一度見たい」というニーズに対応します。

### 4-5. ローカルフォルダ直接検索

Windowsフォルダパス（`C:\`、`X:\` 等のネットワークドライブ含む）を登録するだけで、深掘り検索時にフォルダ内ファイルをAIが直接読み取る。事前のアップロードやベクトル化は不要。

```
入力: フォルダパス「X:\98_PERSONAL\佐野\技術」を登録
  ↓
処理: 検索時にAIがファイル一覧取得 → 関連ファイルを選択 → その場でテキスト変換
  ↓
出力: PDF/Excel/Wordの中身をAIが直接読み、回答を生成
```

- 対応形式: PDF, Excel, Word, PowerPoint, CSV, Markdown, テキスト, JSON, HTML
- 最大100件のファイルをスキャン（100件以上の場合は表示で通知）
- 検索時にオンデマンドでファイル変換（セッションキャッシュで同一ファイルの再変換を防止）

### 4-6. AIエージェント検索（深掘りモード）

AIが人間のように「一覧を見る → 検索する → 読む → 考える」を自律的に繰り返す。フォルダ内ファイルも含めて探索する。

```
ユーザー: 「K298C03573のEGR-Vのエア圧力計の規格値は？」
  ↓
AI Step 1: list_documents でファイル一覧を確認（フォルダファイル含む）
AI Step 2: 「圧力計」「規格値」で search_in_document → 3件ヒット（部分一致）
AI Step 3: キーワードを絞って「圧力計」のみで再検索 → 10件ヒット
AI Step 4: read_document_section で該当セクションを読み込み
AI Step 5: 規格値「0.45±0.05 MPa」を発見、回答を生成
```

- 最大10回の反復（設定画面で1〜15回に調整可能）
- スライディングウィンドウ検索: 複数キーワードが異なる行にあっても20行以内ならヒット
- ORフォールバック: AND検索で0件の場合、部分一致結果を返して再検索のヒントを提供

### 4-7. 外部システム連携

- **GitHub連携**: パブリックリポジトリのMarkdown/テキストファイルを差分同期
- **Gitea連携**: 社内Giteaサーバーのリポジトリからファイルを自動同期
- **外部API**: APIキー認証付きの同期チャットAPIで、Slack Bot等からRAG検索を利用可能

## 5. データソース統合と検索フロー

```
データソース                          RAG Phantom
┌──────────────┐
│ ファイルアップ │  ──ベクトル化──→  ┌────────────────────────┐
│ ロード        │                  │                        │
├──────────────┤                  │  ナレッジベース          │
│ ローカル      │  ──直接読取り──→  │                        │
│ フォルダ      │   (オンデマンド)   │  ┌────────────────┐    │
├──────────────┤                  │  │ 通常: ベクトル検索│    │
│ GitHub /     │  ──同期──────→  │  │ 深掘: AI自律検索 │    │
│ Gitea        │                  │  └────────────────┘    │
└──────────────┘                  └────────────────────────┘
```

### 通常モード（ベクトル検索）

```
ユーザー質問 → クエリ解析 → ユーザー辞書変換 → クエリ拡張(3件)
  → Qdrant検索(密ベクトル+スパース) → RRF統合 → リランク → Claude回答生成(SSE)
```

### 深掘りモード（AIエージェント検索）

```
ユーザー質問 → AIが自律的にツールを使い分け:
  list_documents   → ファイル一覧確認（フォルダファイル含む）
  search_in_document → キーワード検索（スライディングウィンドウ）
  read_document_section → 該当箇所を詳細に読み込み
  → 最大10回反復 → 根拠に基づく回答生成
```

## 6. 実装済みの価値

| 価値 | 詳細 |
|---|---|
| **検索時間の短縮** | フォルダ探索不要。自然言語で質問するだけで関連情報をAIが抽出・要約 |
| **フォルダ指定の手軽さ** | パスを登録するだけ。アップロードもベクトル化も不要で即座に検索可能 |
| **AIの自律的な検索** | 深掘りモードでAIがファイル選択→検索→読み込みを自動で繰り返す |
| **表記ゆれの解消** | ユーザー辞書＋セマンティック検索で、正式名称を知らなくても正確にヒット |
| **知識の属人化防止** | 個人PCの文書をナレッジベースに集約。誰でも同じ品質の回答を得られる |
| **パーソナライズ回答** | ユーザーメモリが所属・担当を記憶し、文脈に応じた回答を生成 |
| **12形式の文書対応** | PDF・Excel・Word・PowerPoint・CSV・画像など、現場で使われる主要形式をすべて取り込み可能 |
| **回答の根拠明示** | 参照元ドキュメントを一覧表示し、プレビューで内容を確認可能 |

## まとめ

1. 社内文書の散在・検索困難を**ベクトル検索＋フォルダ直接検索＋AIエージェント検索**で解決する仕組みを構築済み
2. フォルダパス指定とベクトル化の**2つのアプローチ**で、手軽さと精度を両立
3. その先に**「現場の誰もが、必要な知識に即座にアクセスできる」**世界を実現する

> **RAG Phantom**
> 社内の知識を幽霊（Phantom）のように — 見えないけれど、呼べばすぐそこにいる
